## 2️⃣ Core Philosophy & Concepts (Non-Negotiable)

### 🔒 One-Time Build (OPOJB / OPOJD)
- A unit of work is built **once**
- No partial progress states
- No “almost done”, “93%”, or “flaky but OK”
- Only terminal states are allowed:
  - **COMPLETE**
  - **BLOCKED**
- Anything else = governance failure

### 🧭 True North
- Governance correctness > speed
- Auditability > convenience
- Structural prevention > human memory

### 🌊 Ripple Effect
All changes must **ripple downward**:
Governance → FM Contract → Builder Contracts → Execution

markdown
Copy code
Nothing is assumed to propagate automatically.

### 🧠 AI Autonomy (With Accountability)
- FM is autonomous execution authority
- FM authors plans, issues, and instructions
- Builders execute strictly under FM
- Governance agents analyze and ratchet learning
- Automation assists awareness, **never replaces authority**
