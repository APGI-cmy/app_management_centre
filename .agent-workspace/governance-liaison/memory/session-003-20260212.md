## Lessons

### What Worked Well

1. **Evidence-First Error Messages**
   - Every validation step includes exact path, schema, remediation, authority
   - No log archaeology required - failures are self-explanatory
   - Reduces debugging time for future PRs

2. **Dynamic Agent Detection**
   - Using `find` to discover agent workspaces works across all agent types
   - Prevents hardcoding issues that caused PR #740 gap
   - Future-proof for new agent types

3. **Layered Validation**
   - Multiple validation steps catch different aspects
   - Each step is focused and testable independently
   - Clear separation of concerns (wake-up, session, learning, etc.)

4. **Code Review Tool Integration**
   - Caught subtle issue (invalid condition) that could cause runtime problems
   - Immediate fix prevented technical debt
   - Validates importance of automated review

5. **Evidence Artifact Bundle**
   - Complete bundle structure enforces documentation discipline
   - Clear audit trail for governance changes
   - Supports future RCA if issues arise

### What Was Challenging

1. **Agent-Agnostic Logic Complexity**
   - Challenge: Making validation work for any agent workspace layout
   - Solution: Used `find` with pattern matching
   - Learning: Dynamic discovery adds complexity but provides flexibility

2. **Ephemeral File Validation**
   - Challenge: working-contract.md is gitignored, can't check directly
   - Solution: Validate mention in persistent prehandover proof
   - Learning: For ephemeral artifacts, validate evidence of creation, not the artifact itself

3. **Balancing Strictness vs. Usability**
   - Challenge: Making learning artifacts blocking could cause friction
   - Solution: Clear error messages with exact remediation steps
   - Learning: Strict enforcement needs excellent error messages

4. **Code Review Condition Logic**
   - Challenge: Incorrect condition `steps.validate-gate-schema.outputs != ''`
   - Solution: Simplified condition, removed unnecessary check
   - Learning: GitHub Actions output objects are always truthy; check specific fields or outcome
