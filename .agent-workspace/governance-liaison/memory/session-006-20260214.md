### What Future Sessions Should Know
1. **GitHub repository_dispatch uses underscores** - Not hyphens. Convention: `event_name` not `event-name`
2. **Event type matching is exact** - Case-sensitive, character-by-character comparison
3. **Workflow event mismatches are silent** - No error messages, check character-by-character
4. **Always validate against canonical source** - Don't assume, verify sender/receiver alignment
5. **Reference implementations are debugging gold** - Find working setup, compare configurations
6. **Small diffs can be big fixes** - One character difference caused complete integration failure
7. **Documentation consistency matters** - Update all references when fixing implementation

## Debugging Protocol (Reusable Pattern)

**For Cross-Repository Integration Issues:**

1. **Verify receiver configuration exists and is correct**
   - Check workflow file exists
   - Validate event trigger configuration
   - Confirm workflow is enabled

2. **Check canonical sender configuration**
   - Fetch sender workflow from canonical repo
   - Extract exact event type being sent
   - Note any payload structure

3. **Character-by-character comparison**
   - Compare event types exactly
   - Check for hyphen vs underscore
   - Verify case sensitivity

4. **Validate against reference implementation**
   - Find known working implementation
   - Compare configurations
   - Align with proven setup

5. **Check platform conventions**
   - Review GitHub API documentation
   - Verify naming conventions
   - Confirm payload structure

6. **Test and document**
   - Validate syntax
   - Create evidence artifacts
   - Document lessons learned
