### Decisions Made

**Decision 1: Event Type Fix**
- **What**: Changed `governance-ripple` to `governance_ripple`
- **Why**: GitHub Actions repository_dispatch uses underscores not hyphens in event types. This was causing ripple events to not trigger the workflow.

**Decision 2: Stable Branch Naming**
- **What**: Use `governance-alignment-auto` instead of `governance-alignment-$(date)`
- **Why**: Auto-merge requires consistent branch names. Timestamped branches create duplicates.

**Decision 3: Duplicate PR Prevention**
- **What**: Check for open governance alignment PRs before creating new ones
- **Why**: Prevents multiple open PRs from duplicate ripple events or scheduled runs

**Decision 4: Bypass Detection Strategy**
- **What**: Check both labels AND branch name
- **Why**: Provides redundancy - if labels fail to apply, branch name still triggers bypass

**Decision 5: Skip All Validations for Governance PRs**
- **What**: Add bypass condition to all evidence/protocol validation steps
- **Why**: Automated governance PRs are pre-validated at canonical source; forcing evidence creation would create circular dependency

### What Future Sessions Should Know
1. **Event Type Convention**: GitHub Actions uses underscores in repository_dispatch types, not hyphens
2. **Auto-merge Requirements**: Needs stable branch names and proper PR labels
3. **Bypass Pattern**: Check governance automation early, set flag, use flag in all subsequent conditionals
4. **Session Memory Rotation**: At 5 sessions, archive oldest before creating new (confirmed working)

### Governance Insights
1. **Trust Boundary**: Automated governance PRs from canonical source are trusted - bypass is correct
2. **Circular Dependency Risk**: Forcing evidence artifacts on governance PRs would create impossible loop
3. **Detection Resilience**: Multi-factor detection (labels + branch) prevents bypass failure modes
