## Critical Pattern Identified: Embedded Templates vs. References

**Pattern Name**: Template Embedding Anti-Pattern

**Observed**: Session 002 (2026-02-12), PR #747

**Context**: Documenting 9-component Living Agent System v6.2.0 template for agent-factory protocol

**Symptom**: File size >30,000 characters (blocks GitHub UI selectability)

**Common Culprits**:
- Component 2: Requirement Mappings (if embedding all 56 requirements)
- Component 3: Validation Hooks (if embedding all 5 hook specs)  
- Component 8: Canonical References (if enumerating all canons)

**Detection**:
```bash
# Check file size
wc -m < .github/agents/<file>.md

# If >30,000 → BLOCKING failure
# If >25,000 → WARNING (buffer too small)

# Check for embedded templates
grep -A 50 "^#### Component" <file>.md | grep -c "^- REQ-" # Component 2
grep -A 30 "^#### Component 3" <file>.md | grep -c "^### VH-" # Component 3  
grep -A 50 "^#### Component 8" <file>.md | grep -c "^- \`" # Component 8
```

**Root Cause**: Attempting to provide complete examples within agent file instead of referencing canonical governance

**Response Protocol**:
1. **STOP** and calculate character count: `wc -m < <file>`
2. **IDENTIFY** embedded templates (look for components with 50+ lines)
3. **REPLACE** with 5-line references:
   ```markdown
   #### Component X: <Name>
   
   **Template source**: `.governance-pack/path/to/canonical.md`
   
   **Required**: [Brief description of what must be included]
   
   **See**: [Reference to canonical location for complete details]
   ```
4. **RE-VALIDATE** character count (<30K required, <25K recommended)
5. **ADD** Step 4.5 character count validation to execution steps
6. **ENFORCE** 30K limit in YAML frontmatter (`file_size_limit`)

**Prevention**:
- Always use references to canonical governance
- Never embed templates >10 lines
- Run `wc -m` before creating PR
- Target <25,000 characters (20% buffer)

**When**: Creating agent-factory protocol documentation for 9-component template

**Instead of**:
```markdown
