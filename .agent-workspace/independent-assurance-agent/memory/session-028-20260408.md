## Learning Notes

1. **Pre-Brief mode is a distinct session type** — no assurance checks executed, no verdict issued. Session memory should record `session_mode: PRE-BRIEF` to distinguish from standard assurance sessions in future session count reviews.
2. **Systemic blocker promotion at Pre-Brief time is more efficient** than discovering it at Phase 3 invocation — if the pre-brief declares it explicitly, governance-liaison has advance warning and can resolve it before triggering a REJECTION-PACKAGE cycle.
3. **GOVERNANCE_CANON_MANIFEST listing a file that doesn't exist** is active, detectable governance drift. IAA should check GOVERNANCE_CANON_MANIFEST file listings against actual `governance/canon/` directory contents at each CANON_GOVERNANCE invocation as a standing check.
4. **Branch name discrepancy** (user-declared vs. actual HEAD) should be flagged at Pre-Brief time rather than discovered mid-assurance — this prevents unnecessary re-invocation cycles.
5. **Canonical commit identification is a prerequisite that cannot be verified by IAA** — it requires CS2 input or governance-liaison sourcing from the canonical repo directly. Future pre-briefs should include a mandatory "canonical commit resolved" field.
