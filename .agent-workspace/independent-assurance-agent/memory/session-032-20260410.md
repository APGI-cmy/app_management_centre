## Learning Notes

1. **Pre-Brief Task Carry-Through**: Wave-1 Pre-Brief Task 4 explicitly declared the Ripple Assessment section as a required fix. CodexAdvisor did not carry this through to session-016. IAA should flag in session memory whenever a known Pre-Brief task requirement is found missing — this strengthens the "known-and-declared" weight of the finding in the REJECTION-PACKAGE.

2. **Session memory `iaa_invocation_result` pre-population pattern**: CodexAdvisor's session-016 session memory pre-populated `iaa_invocation_result: IAA-session-016-wave1-20260410-PASS` before IAA ran. Under A-029, PREHANDOVER proof's `iaa_audit_token` is legitimately pre-populated with the expected reference format. But the session memory's `iaa_invocation_result` field is supposed to capture the ACTUAL result. Pre-populating this before IAA invocation creates an integrity gap — especially when IAA issues REJECTION-PACKAGE, leaving the field as a false statement. Consider: should A-029's expected-reference pre-population pattern be explicitly restricted to PREHANDOVER proof only (not session memory result fields)? This observation should be surfaced to CS2 for a potential new FAIL-ONLY-ONCE rule.

3. **Escalation inbox pre-approval vs. application PR**: IAA session-005 pre-approved the governance liaison's layer-down staging PR (`copilot/layer-down-governance-changes`, CANON_GOVERNANCE category). The current PR (`copilot/apply-codexadvisor-agent-v4-0-2`, AGENT_CONTRACT category) is a distinct invocation. Both are correctly handled as separate IAA invocations. The substantive contract content was verified in session-005; this session verifies the ceremony of the application.

4. **SCOPE_DECLARATION discipline for CodexAdvisor PRs**: CodexAdvisor has a pattern of not updating SCOPE_DECLARATION.md when it creates its governance ceremony artifacts on a fresh branch rebased from main. The SCOPE_DECLARATION.md from main carries forward and is not updated for the new PR's scope. This is an easy fix but has occurred in multiple CodexAdvisor sessions. Consider whether a pre-IAA commit gate checklist item for CodexAdvisor should explicitly include SCOPE_DECLARATION.md update.
