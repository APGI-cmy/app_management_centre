## Learning Notes

1. **Canonical source CANON_INVENTORY stale entries**: When updating CANON_INVENTORY from a canonical source, the source's own inventory may have stale entries for files that WERE updated in the same canonical commit. The GOVERNANCE_ALIGNMENT_INVENTORY.json (computed from actual installed files) should be cross-referenced against CANON_INVENTORY entries to catch any inherited stale hashes.

2. **GOVERNANCE_ALIGNMENT_INVENTORY as ground truth**: When GOVERNANCE_ALIGNMENT_INVENTORY.json and CANON_INVENTORY.json disagree on a file's version/hash, and the installed file matches GOVERNANCE_ALIGNMENT_INVENTORY, CANON_INVENTORY is likely wrong. Cross-check should be a standard step in layer-down execution.
