## Key Decisions

### D-001: WRCC-001 Checker Runs Before QP PASS (Not Just Before PR-Open)
**Decision**: The WRCC pre-PR checker (WRCC-001 §5) is specified to run BEFORE QP PASS is declared,
not only before PR-open.
**Rationale**: If coherence violations are found after QP evaluation, the QP evaluation step must
be repeated. Requiring the coherence check before QP avoids this wasted evaluation cycle.

### D-002: `wrcc_pre_pr_checker_verdict` Field Added to §3c
**Decision**: A new `wrcc_pre_pr_checker_verdict` field is added to the §3c YAML block.
**Rationale**: This creates a machine-parseable record of the checker result without requiring
changes to any existing §3c field semantics. PHCP-001 and EWCS-001 are amended to require
this new field.

### D-003: Additive Design — No Existing Canon Rules Changed
**Decision**: All amendments to EWCS-001 and PHCP-001 are purely additive. No existing rules,
fields, or violation classes were removed or modified.
**Rationale**: Governance learning retention (A-006) requires that existing rules remain intact.
WRCC-001 adds new detection patterns but does not weaken any existing gating rule.
